深度解構：什麼是遞迴？

在編程中，遞迴 (Recursion) 是指一個子程序 (Subroutine) 在執行過程中直接或間接地呼叫自己。這是一種將大問題拆解為較小規模、結構相同的子問題 (Sub-problems) 的解題策略。

要撰寫正確的遞迴函數，必須具備兩個核心元素：

基層情況 (Base Case)

終止遞迴的條件。當滿足此條件時，函數不再呼叫自己，而是返回一個確定的值。若缺失此條件，將導致無限迴圈。

遞迴步驟 (Recursive Step)

將問題簡化，並傳入較小的參數再次呼叫自身。每一次的遞迴呼叫，都必須讓狀態向「基層情況」靠近。

互動模擬：階乘 (Factorial)

計算公式：$$n! = n \times (n-1)!$$，基層情況：$$1! = 1$$

重置模擬 (Reset) 


def factorial(n):


    if n == 1: # Base Case


        return 1


    # Recursive Step


    return n * factorial(n - 1)



int factorial(int n) {


    if (n == 1) { // Base Case


        return 1;


    }


    // Recursive Step


    return n * factorial(n - 1);


}

常見迷思 (Common Myths)

迷思：遞迴總是比使用 for/while 迴圈 (Iteration) 更好？

事實： 並非如此。雖然遞迴在處理樹狀資料（Tree）或圖形（Graph）時程式碼更簡潔易讀，但因為每次呼叫都需要在記憶體中建立 Stack Frame，其 空間複雜度 (Space Complexity) 較高。如果遞迴過深，會導致記憶體耗盡，發生 堆疊溢出 (Stack Overflow) 錯誤。許多簡單問題使用迭代 (Iteration) 在效能上更佳。

概念總結 (Key Takeaways)

  • 遞迴本質上是利用電腦的 調用堆疊 (Call Stack) 來暫存狀態。
  • 必須包含 基層情況 (Base Case) 作為「煞車系統」，否則程式會崩潰。
  • 遞迴過程分為兩階段：「遞進 (Winding)」時不斷呼叫並將參數推入堆疊；觸及基層情況後開始「回歸 (Unwinding)」，將計算結果依序返回並彈出堆疊。

思考與連結 (Think & Link)

Q: 如果我們在計算 factorial(3) 時，忘記寫 `if (n == 1)` 這個基層情況，電腦底層會發生什麼事？

A: 函數會無止盡地呼叫 `factorial(0)`, `factorial(-1)`, `factorial(-2)`...。每次呼叫都會佔用一塊調用堆疊 (Call Stack) 的記憶體空間。由於記憶體是有限的，最終電腦會耗盡配置給堆疊的記憶體空間，並拋出 堆疊溢出 (Stack Overflow) 的執行期錯誤 (Runtime Error)，導致程式強行終止。
